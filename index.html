Every pattern in this AUTOSAR C programming book addresses a real class of defect present in production automotive firmware — not invented lab scenarios.

🔴
Unchecked RTE Return Values

Rte_Read and Rte_Write can fail silently. Processing stale or invalid signal values without checking the return code causes undefined system behaviour.

Rte_Read_Signal(&val);
/* return ignored → UB on failure */
🔴
Integer Overflow in Signal Scaling

Silent overflow in uint8/uint16 arithmetic produces wrong physical values with no compiler warning — wrong actuator output, MISRA Rule 10.x violations.

uint8_t s = (raw * 3) / 2;
/* overflow if raw > 170 */
🔴
Stack Overflow Across OS Tasks

AUTOSAR OS stacks are static. Deep call chains or unexpected ISR nesting silently corrupt adjacent task stacks — no immediate fault, random behaviour later.

/* Miscalculated stack depth */
/* corrupts neighbour → random faults */
🔴
Race Conditions Without SchM

Shared globals accessed across Tasks and ISRs without SchM_Enter/Exit exclusion areas cause data inconsistency impossible to reproduce in testing.

globalFlag = TRUE;
/* no exclusion area → torn read */
🔴
NULL Pointer in BSW Callbacks

Com, CanIf, and NvM callbacks receive buffer pointers that can be NULL on initialisation sequences or error paths — dereferencing causes hard faults not seen in SIL.

*pData = value;
/* pData not validated → hard fault */
🔴
Bitfield Undefined Behaviour

Bitfield manipulation in C has implementation-defined and undefined behaviours that MISRA flags but engineers dismiss — until a compiler upgrade silently changes behaviour.

struct.bit = 1u << 8;
/* UB: implementation-defined padding */

What defensive C looks like
applied to AUTOSAR

Each chapter shows the unsafe pattern, explains why it fails in an AUTOSAR production context, and gives the correct defensive replacement.

✗ Unsafe Pattern
/* Dangerous: no return check */
uint16 sensorVal;

Rte_Read_SensorInput(&sensorVal);

if(sensorVal > THRESHOLD) {
  ActivateActuator();
}

/* Dangerous: overflow risk */
uint8 scaled;
scaled = (sensorVal * 5u) / 2u;
/* overflows if sensorVal > 51 */
✓ Defensive Pattern
/* Safe: validate RTE return */
uint16 sensorVal = 0u;
Std_ReturnType rteRet;

rteRet = Rte_Read_SensorInput(&sensorVal);
if(rteRet != RTE_E_OK) {
  Dem_SetEventStatus(SENSOR_ERR, FAILED);
  return;
}
/* Safe: promote before multiply */
uint32 tmp = (uint32)sensorVal * 5u;
uint8  scaled = (uint8)(tmp / 2u);
